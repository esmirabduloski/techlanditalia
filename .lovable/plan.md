
The user is asking why Google Search Console isn't indexing all pages, and whether the sitemap is the issue. They want ALL URLs included, especially blog articles.

Looking at the current `public/sitemap.xml`, I can see it's a **static** file that only contains hardcoded URLs:
- Homepage, /corsi, individual course pages (hardcoded), /prenota, /blog, /faq, /chi-siamo, /contatti, /lavora-con-noi, /privacy, /termini, /cookie, /accessibilita

**Critical issues identified:**
1. **Blog articles are NOT included** — only the `/blog` index page is listed. Every individual `/blog/:slug` is missing.
2. **Landing pages `/lp/:slug` are NOT included** — the dynamic landing page system creates pages stored in DB.
3. **Static file**: Since the sitemap is hardcoded in `public/sitemap.xml`, every new blog post or landing page requires manual updates — which never happens, so Google can't discover them.
4. **Last modified dates are stale** (2026-03-30) — Google deprioritizes sitemaps that don't reflect real updates.
5. **Missing course pages**: e.g., `python-ai` (the actual slug for Python PRO & AI per memory) might not match what's in the sitemap (currently `python-pro-ai`).
6. **`/accessibilita` page exists but might be missing** — let me check... actually it IS missing from the sitemap.

**Solution: dynamic sitemap generation**

Best approach for a Vite SPA with Supabase backend: create a **Supabase Edge Function** (`generate-sitemap`) that:
- Queries the `blog_posts` table for all published articles
- Queries the `landing_pages` table for all active landing pages
- Queries the `courses` table for all visible courses (slugs)
- Combines with hardcoded static pages
- Returns valid XML with `Content-Type: application/xml`

Then update `public/robots.txt` and the sitemap reference to point to the edge function URL.

**Alternative**: Keep `public/sitemap.xml` but make it an **index sitemap** that references multiple sub-sitemaps (one for blog, one for courses, one for landing) generated by edge functions. This is overkill for the size — single dynamic sitemap is enough.

**Why pages aren't indexed (broader context):**
Beyond the sitemap, Google indexing depends on:
- Internal linking (blog posts linked from /blog index? landing pages?)
- Crawlability (SPA — content rendered client-side; Google does render JS but it's slower)
- Quality signals
- robots.txt (already correct, allows everything except /admin and /area-riservata)
